Project | 01 Heap Allocator
  • This is the final project for the popular course CS107, taught at Stanford. I built a heap allocator; implemented the functions malloc, free and realloc.
  • I designed the allocator using segragated linked lists of free memory. I divided free blocks in three group sizes: small, medium and large and stored in a table, giving it a dynamic design. When memmory was requested, the allocator would first check the cached blocks before extending the heap. If a cached block was too big compared to the requested size, the allocator would split it, but before adding the remaining to the table, it checked if its left and right neighbors were free to coalesce them.
  • Each free block contained a header and a footer structure to store size and status. This way the allocator would know the exact location of the contiguous neighbors before coalescing them
  • The most important lessons I learned on this project was how the heap works. Similarly, I developed a strong foundation on the use of pointers and pointer arithmatic. However, I would say the most valuable lessons was "paying attention to small details and staying consistent with my code" and in order to stay consistent, first I must spend time designing on paper, before writing any code. This has been one of the most challenging, yet most rewarding projects I have worked on

Project | 02 Unagi Task Manager
  • I built a Task Manager for Android. I developed this app while at an Android BootCamp at Twitter headquarters (Thank you much Twitter!). Unagi's purpose is to help individuals stays productive with the help of a social twist.
  • Whenever the user posts a task, this gets posted to her Twitter account, holding the person accountable for it. When the task is either finished or not completed on time, the app posts to the user's Twitter. There is no an option to delete tasks.
  • I used Twitter's Fabric API for the login portion as well as for the posting. Similarly, for the list of activities I used RecyclerView instead of ListView.
